A lib "reportLib"

A lib possibilita a geração de relatório orientado ao padrão estético/funcional do SEI. Esse processo demanda a prévia construção do script do modelo do relatório, no formato JSON, e das variáveis "reportInfo", "dataOverview" e "callingApp".
  • "reportInfo": Trata-se da variável que armazena informações gerais, aplicáveis a todos os fluxos de dados sob análise.
  • "dataOverview": Trata-se da variável que armazena informações específicas de cada fluxo de dados sob análise.
  • "callingApp": Trata-se de handle para a aplicação que aciona a lib, possibilitando o uso das informações constantes em suas propriedades públicas e nos seus componentes na geração de imagens e tabelas, por exemplo.
Como saída, uma string com o código fonte do HTML.
% Sintaxe:
htmlReport = report.Controller(reportInfo, dataOverview, callingApp);
Atualmente, a lib possibilita a geração dos seguintes tipos de componentes:
  • ItemN1: classe "Item_Nivel1"
  • ItemN2: classe "Item_Nivel2"
  • ItemN3: classe "Item_Nivel3"
  • Paragraph: classe "Texto_Justificado"
  • Footnote: classe "Tabela_Texto_8", cor "#808080"
  • List: classe "Texto_Justificado"
  • Image: classe da legenda "Tabela_Texto_8"
  • Table: classe da legenda "Tabela_Texto_8", classe da tabela "tabela_corpo"

As variáveis de entrada

reportInfo

Trata-se da variável que armazena informações gerais, aplicáveis a todos os fluxos de dados sob análise. É uma estrutura que possui ao menos os campos "Model" e "Function", mas há funcionalidades definidas para os campos opcionais "Path" e "Version". Outros campos podem ser criados para organizar a informação sob análise. Informações específicadas de cada de fluxo de dados sob análise devem ser armazenadas em "dataOverview".
%% Estrutura vazia composta apenas pelos campos OBRIGATÓRIOS:
reportInfo = struct('Model', {}, 'Function', {});
 
%% Exemplo 1
% Estrutura simples composta apenas pelos campos OBRIGATÓRIOS:
reportInfo = struct('Model', struct('Version', 'preview', ... % versão do relatório: 'preview' ou 'final'
'Name', 'Template 001/2024'), ... % nome do script definido no arquivo "html_General.cfg", da subpasta "Template"
'Function', struct('var_Issue', '123456')); % número da issue no Fiscaliza (solicitação, ação ou inspeção)
 
%% Exemplo 2
% Estrutura complexa, que inclui campos OPCIONAIS (além dos supracitados campos OBRIGATÓRIOS):
reportInfo = struct('Path', struct('userPath', 'D:\ANATEL'), ... % local onde serão salvas as imagens eventualmente geradas pela lib
'Version', struct('appModule1', struct('name', 'appModuleName1', 'release', '10/12/2023', 'version', '0.04'), ... % informações relacionadas a um módulo hipotético "appModuleName1" que suportaria a solução
'appModule2', struct('name', 'appModuleName2', 'release', '31/01/2024', 'version', '0.10')), ... % informações relacionadas a um módulo hipotético "appModuleName2" que suportaria a solução
'Model', struct('Version', 'preview', ...
'Name', 'Template 001/2024'), ...
'Function', struct('var_Issue', '123456', ...
'var_Receiver', 'analyzedData.InfoSet.Receiver', ... % variável "Receiver"
'var_FreqStart', 'analyzedData.InfoSet.FreqStart', ... % variável "FreqStart"
'var_FreqStop', 'analyzedData.InfoSet.FreqStop', ... % variável "FreqStop"
'var_Samples', 'numel(analyzedData.InfoSet.Array)', ... % variável "Samples"
'var_Locations', 'strjoin(analyzedData.InfoSet.Location, '', '')', ... % variável "Locations"
'img_Pumpkin', 'example.fcn.imgPumpkin(reportInfo, analyzedData, callingApp, imgSettings)', ... % gerador da imagem "Pumpkin"
'img_Surf', 'example.fcn.imgSurf(reportInfo, analyzedData, callingApp, imgSettings)', ... % gerador da imagem "Surf"
'img_Ordinary1', 'example.fcn.imgOrdinary(reportInfo, analyzedData, callingApp, imgSettings)', ... % gerador da imagem "Ordinary"
'tbl_Ordinary1', 'example.fcn.tableOrdinary(reportInfo, analyzedData, callingApp, tableSettings)')); % gerador da tabela "Ordinary"

dataOverview

Trata-se da variável que armazena informações específicas de cada fluxo de dado sob análise. É uma estrutura que possui ao menos os campos "ID", "InfoSet" e "HTML". Outros campos podem ser criados para organizar a informação sob análise. Informações gerais, que se aplicam a todos os fluxos de dados, devem ser armazenadas em "reportInfo".
  • "ID": Identificação textual do fluxo de dados sob análise.
  • "InfoSet": Campo que armazena informações dos dados brutos e da sua análise, os quais podem ser encapsulados de qualquer forma que faça sentido para o desenvolvedor - "cell", "struct", "table", matriz numérica, instância de uma classe etc.
  • "HTML": Estrutura fixa, com os campos "Component", "Source" e "Value", que armazena informações que podem compor o relatório e foram geradas anteriormente ao relatório.
%% Estrutura vazia composta apenas pelos campos OBRIGATÓRIOS:
dataOverview = struct('ID', {}, 'InfoSet', {}, 'HTML', {});
 
%% Exemplo 1
% Neste exemplo, considere que os dados brutos e a sua análise foram organizados numa variável chamada "specData". E considere que, anteriormente à geração do relatório, foram criadas as imagens e tabelas "T11" e "I11"
% (relacionadas ao fluxo de dados 1), "T21" e "I21" (relacionadas ao fluxo de dados 2) e "T31", "T32" e "I31" (relacionadas ao fluxo de dados 3). Perceba que "T32" não é uma tabela, mas uma estrutura que armazena o path

callingApp

Trata-se de handle para a aplicação que aciona a lib. Caso a lib seja acionada diretamente do prompt do MATLAB, pode-se omitir esse argumento.

O script

O script do modelo do relatório, no formato JSON, guia a lib no processo de geração de um relatório. Trata-se de estrutura hierarquica baseada em componentes HTML, organizados de forma a representar diferentes seções e tipos de conteúdo do relatório.

ItemN1

  • Type: "ItemN1"
  • Recurrence: 0 | 1 (em sendo indicado 1, a lib executará a seção numel(dataOverview) vezes)
  • Data:
  • Editable: "true" | "false"
  • String: Título da seção.
  • Children: Um objeto ou array de objetos representando os componentes filhos deste "ItemN1", detalhando o conteúdo da seção.

ItemN2 e ItemN3

  • Type: "ItemN2" ou "ItemN3", representando uma subseção de um "ItemN1" ou "ItemN2".
  • Data:
  • Editable: "true" | "false"
  • String: Título da subseção. Pode ser uma string formatada, contendo "%s", por exemplo, de forma que a lib substitua esse trecho da string pelo valor de uma variável.
  • Settings:
  • Source: Nome da variável.
  • Precision: Formato da variável ("%s" para string, "%.0f" para valor numérico sem casas decimais e "%.3f MHz" para valor numérico com três casas decimais e com unidade de medida igual a MHz, por exemplo).
  • Multiplier: Fator que será multiplicado ao valor da variável, caso se trate de um número. Não aplicável para variáveis que são strings.
{
"Type": "ItemN1",
"Recurrence": 0,
"Data": {
"Editable": "false",
"String": "OBJETIVO",
"Children": {
"Type": "ItemN2",
"Data": {
"Editable": "false",
"String": "Atendimento à demanda de fiscalização cadastrada no Fiscaliza sob a <b>Atividade de Inspeção nº %.0f</b>, relacionada à monitoração realizada na(s) localidade(s) <b>%s</b>.",
"Settings": [
{
"Source": "var_Issue",
"Precision": "%.0f",
"Multiplier": 1
},
{
"Source": "var_Location",
"Precision": "%s",
"Multiplier": -1
}
]
}
}
}
}

Paragraph e Footnote

  • Type: "Paragraph" | "Footnote"
  • Data: Um objeto com os campos "Editable", "String" e "Settings", funcionalmente idênticos ao descrito anteriormente para os componentes "ItemN2" e "ItemN3".
{
"Type": "Paragraph",
"Data": {
"Editable": "false",
"String": "Trata-se de monitoração conduzida pelo sensor %s. A análise dessa monitoração - que contempla %.0f varreduras - engloba a aferição da ocupação dos <i>bins</i>, a detecção e classificação de emissões e a geração de uma representação gráfica.",
"Settings": [
{
"Source": "var_Receiver",
"Precision": "%s",
"Multiplier": -1
},
{
"Source": "var_Samples",
"Precision": "%.0f",
"Multiplier": 1
}
]
}
}

Image

  • Type: "Image"
  • Data:
  • Origin: "FunctionEvaluation" | "DataProperty"
  • Source: Identificação da imagem. Caso a origem seja "FunctionEvaluation", este valor deverá compor a estrutura reportInfo.Function, sendo um campo. Caso seja "DataProperty", este valor deverá ser "Source" de um dos componentes da estrutura dataOverview.HTML.
  • Layout: Campo de uso livre, que pode ser usado no processo de criação da imagem.
  • Caption: Legenda da imagem.
  • Settings: Configurações da imagem, como largura e altura.
  • Width: "50%" | "auto"
  • Height: "50%" | "auto"
  • LineBreak: 0 | 1 (indica se deve haver uma quebra de linha após a imagem)
  • Intro: Um JSON serializado descrevendo componente HTML textual, como "ItemN2" ou "Paragraph", por exemplo, o qual será apresentado antes da imagem. Caso não seja necessário incluir essa mensagem textual de introdução, pode-se preencher esse campo como "".
  • Error: Um JSON serializado descrevendo componente HTML textual, como "ItemN2" ou "Paragraph", por exemplo, o qual será apresentado caso ocorra algum problema no processo de inclusão da imagem. Caso não seja necessário incluir essa mensagem textual de erro, pode-se preencher esse campo como "".
{
"Type": "Image",
"Data": {
"Origin": "DataProperty",
"Source": "img_Ordinary1",
"Layout": "",
"Caption": "A legenda da imagem...",
"Settings": {
"Width": "25%",
"Height": "auto"
},
"LineBreak": 1,
"Intro": "{\"Type\":\"ItemN2\",\"String\":\"Esse texto é uma introdução à imagem apresentada a seguir...\"}",
"Error": "{\"Type\":\"ItemN2\",\"String\":\"Esse texto é uma mensagem de erro à não inclusão da tabela...\"}"
}
}

Table

  • Type: "Table"
  • Data:
  • Origin: "FunctionEvaluation" | "DataProperty"
  • Source: Identificação da tabela. Caso a origem seja "FunctionEvaluation", este valor deverá compor a estrutura reportInfo.Function, sendo um campo. Caso seja "DataProperty", este valor deverá ser "Source" de um dos componentes da estrutura dataOverview.HTML.
  • Columns: Um array de números ou nomes das colunas, definindo quais delas devem ser exibidas. Por exemplo: [1, 2, 4] ou ["Nome da coluna 1", "Nome da coluna 2", "Nome da coluna 4"].
  • Caption: Legenda da tabela.
  • Settings: Configurações específicas para cada coluna da tabela, como precisão, multiplicador (para ajuste de escala dos valores numéricos), e largura.
  • Editable: "true" | "false"
  • String: Título da coluna.
  • Precision: Precisão de dados da coluna:
  • Multiplier: Fator que será multiplicado ao valor da variável, caso se trate de um número. Não aplicável para variáveis que são strings.
  • Width: Largura da coluna, como "80px" ou "auto".
  • LineBreak: 0 | 1 (indica se deve haver uma quebra de linha após a tabela)
  • Intro: Um JSON serializado descrevendo componente HTML textual, como "ItemN2" ou "Paragraph", por exemplo, o qual será apresentado antes da tabela. Caso não seja necessário incluir essa mensagem textual de introdução, pode-se preencher esse campo como "".
  • Error: Um JSON serializado descrevendo componente HTML textual, como "ItemN2" ou "Paragraph", por exemplo, o qual será apresentado caso ocorra algum problema no processo de inclusão da tabela. Caso não seja necessário incluir essa mensagem textual de erro, pode-se preencher esse campo como "".
